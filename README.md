Dabeeo 맞춤형 신안보/공공 입찰공고 인텔리전스 봇
본 프로젝트는 나라장터(G2B)와 국방전자조달(D2B) 시스템을 매일 자동으로 모니터링하여, 다비오(Dabeeo)의 비즈니스 도메인에 최적화된 핵심 사업 기회만을 선별해 담당자에게 이메일로 브리핑해 주는 자동화 파이프라인입니다.

주요 기능 (Features)
G2B & D2B 통합 모니터링: 최근 7일 이내에 등록된 나라장터(용역/물품) 및 국방전자조달(경쟁입찰/공개수의) 미마감 공고를 한 번에 수집합니다.

스마트 노이즈 필터링 (Negative Filter): 단순 제조, 공사, 하드웨어 납품 등 다비오의 업무 영역과 무관한 공고는 30여 개의 강력한 네거티브 키워드를 통해 원천 차단합니다.

다비오 맞춤형 스코어링 (Scoring Algorithm): 수집된 공고의 제목을 분석하여 자사 핵심 역량(위성, 영상, 공간정보, AI 등)과의 연관도 점수를 산정합니다.

매일 아침 자동 브리핑: GitHub Actions의 Cron 스케줄러를 통해 평일(월~금) 아침 08:30에 정제된 HTML 리포트를 자동으로 발송합니다.

스코어링 및 등급 판정 기준
수집된 공고는 내부 알고리즘에 의해 평가되며, 20점 미만의 '하 (단순 키워드)' 등급은 피로도를 줄이기 위해 리포트에서 자동 제외됩니다.

상 (핵심 타겟)
조건: 연관도 점수 50점 이상

필수 허들: 공고명에 위성, 드론, 공간정보 중 최소 1개 이상이 명시적으로 포함되어야만 '상' 등급으로 분류됩니다. (미포함 시 '중'으로 강등)

대응: 다비오의 핵심 사업과 직결될 가능성이 매우 높은 공고이므로 최우선 검토가 필요합니다.

중 (검토 권장)
조건: 연관도 점수 20점 이상 ~ 50점 미만 (또는 50점 이상이나 필수 허들 키워드가 없는 경우)

대응: AI, 영상 분석, 플랫폼 구축 등 다비오의 기술력이 활용될 수 있는 연관 사업이므로 추가적인 상세 규격서 확인을 권장합니다.

리포트 100% 활용 가이드 (FAQ)
Q. 국방전자조달(D2B) 공고는 버튼을 눌러도 바로 안 들어가져요.

국방전자조달 시스템의 강력한 보안 및 세션 정책으로 인해 외부에서 상세 페이지로의 직접 링크 연결이 차단되어 있습니다.
메일 본문에 표시된 검색용 공고번호를 드래그하여 복사한 뒤, D2B 홈페이지의 검색창에 붙여넣기 하시면 가장 빠르게 상세 내용을 확인하실 수 있습니다.

Q. 마감된 공고도 오나요?

본 메일은 발송 시점 기준으로 입찰/참가 등록이 마감되지 않은 '진행 중'인 유효 공고만을 선별하여 전달합니다.

Q. 단순 제조/공사/물품은 안 보이나요?

네, 제조, 공사, 서버, 하드웨어 등 30여 개의 강력한 네거티브 필터링 알고리즘을 거쳐 다비오와 무관한 공고는 전면 자동 제외 처리됩니다.

시스템 커스터마이징 가이드 (수정 방법)
알고리즘 기준이나 수신자를 변경해야 할 경우, 메인 스크립트인 dabeeo_bid_master.py를 직접 수정하여 반영할 수 있습니다.

수신자(이메일) 변경:

코드 상단 1. 시스템 통합 글로벌 설정 영역의 RECEIVERS 리스트를 수정합니다.

예시: RECEIVERS = ['lucas.park@dabeeo.com', 'new.member@dabeeo.com']

제외할 단어(네거티브 키워드) 추가:

코드 중단 2. 다비오 고도화 스코어링 영역의 NEGATIVE_KEYWORDS 리스트에 원치 않는 단어를 추가합니다.

해당 단어가 포함된 공고는 리포트에서 영구 제외됩니다.

가중치 키워드 수정:

높은 점수를 부여할 단어는 HIGH_WEIGHT_KEYWORDS (건당 +30점)에 추가합니다.

중간 점수를 부여할 단어는 MID_WEIGHT_KEYWORDS (건당 +15점)에 추가합니다.

검색 자체에 사용될 기본 키워드는 최상단 KEYWORDS 리스트에서 수정합니다.

실행 및 운영 방법
1. 자동 실행 (Cron Schedule)
본 파이프라인은 GitHub Actions에 의해 매주 평일(월~금) KST 오전 08:30에 자동으로 실행되도록 설정되어 있습니다. (.github/workflows/daily_brief.yml 참고)

2. 수동 실행 (Manual Trigger)
스케줄 외에 즉시 현재 시점의 공고 현황을 리포트로 받아보고 싶을 때는 아래 방법을 사용하세요.

이 리포지토리의 상단 Actions 탭으로 이동합니다.

좌측 워크플로우 목록에서 Dabeeo Unified Bid Notice Cron Engine (또는 daily_brief.yml)을 클릭합니다.

우측에 나타나는 Run workflow 버튼을 클릭하여 강제 실행합니다. (약 1~2분 소요 후 메일 발송)

3. 환경 변수 (Secrets) 셋팅
메일 발송 및 API 통신을 위해 GitHub Repository의 Settings > Secrets and variables > Actions에 다음 값이 등록되어 있어야 합니다.

SMTP_EMAIL: 발송을 담당할 GMAIL 주소 (예: 본인 이메일)

SMTP_PASSWORD: GMAIL 앱 비밀번호 (16자리)
